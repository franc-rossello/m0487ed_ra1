RA13 Diferència els conceptes de codi font, objecte i executable.

Introducció al Desenvolupament d'un Programa Informàtic

En el desenvolupament de programari, un programa passa per diverses etapes abans de poder ser executat per un ordinador. Aquest procés inclou la creació del codi font, la seva conversió en codi objecte i, finalment, la generació d'un fitxer executable. A continuació, s'analitzen aquests tres conceptes en detall.

Codi Font

El codi font és el conjunt d'instruccions escrites per un programador en un llenguatge de programació com C, Java, Python, etc. Aquest codi és llegible per humans i defineix la lògica i funcionalitat d'un programa.

Característiques principals del codi font:

  • És llegible per humans.
  • Pot ser modificat i millorat fàcilment.
  • Es guarda en fitxers amb extensions com .c, .java, .py.

Exemple de codi font en C:


      #include <stdio.h>
      int main() {
          printf("Hola, món!\n");
          return 0;
      }

Codi Objecte

El codi objecte és una versió binària del programa obtinguda després de la compilació del codi font. Tot i que és més proper al llenguatge màquina, encara no es pot executar directament sense ser enllaçat.

Característiques del codi objecte:

  • És un conjunt d'instruccions en llenguatge màquina.
  • Necessita ser enllaçat amb altres fitxers per formar un executable.
  • Es guarda en fitxers amb extensions com .o o .obj.

Procés de compilació:

Quan es compila el codi font d'un programa en C amb el compilador gcc, es genera un fitxer objecte:


      gcc -c programa.c -o programa.o

Fitxer Executable

El fitxer executable és la versió final d'un programa que pot ser executada pel sistema operatiu. Es genera després d'enllaçar el codi objecte amb biblioteques necessàries.

Característiques dels executables:

  • Conté totes les instruccions necessàries per ser executat.
  • El sistema operatiu pot carregar-lo directament.
  • Extensions habituals: .exe en Windows, sense extensió en Linux i macOS.

Generació de l'executable:


      gcc programa.o -o programa




Procés de Desenvolupament d'un Programa Informàtic

El procés de creació d'un programa informàtic segueix diverses etapes que inclouen el desenvolupament, la compilació i l'enllaç. A continuació, es detallen aquestes etapes:

  1. Escriure el codi font: El programador crea el codi font que defineix la lògica del programa.
  2. Compilació: El compilador converteix el codi font en codi objecte. Aquesta etapa detecta errors sintàctics o lògics en el codi font.
  3. Enllaç (Linking): Els arxius de codi objecte es combinen amb llibreries externes per generar un programa complet.
  4. Creació de l'executable: El codi objecte es converteix en un fitxer executable que pot ser executat pel sistema operatiu.

Un cop completat el procés, el programa està llest per ser distribuït i executat pels usuaris.
