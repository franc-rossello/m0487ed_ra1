RA13 - Diferencia els conceptes de codi font, objecte i executable.




Introducció

Els tres components (codi font, codi objecte i arxiu executable) són fonamentals per al desenvolupament de programari. Cada un té un propòsit específic en el procés de transformació de les idees d'un programador en un programa que es pot utilitzar en un ordinador.

El codi font és on comença tot, el codi objecte és una representació intermèdia que es genera durant la compilació, i l'arxiu executable és el producte final que el sistema operatiu pot executar. Comprendre aquests tres conceptes és crucial per als desenvolupadors, ja que els permet entendre com funciona el procés de creació de programari i com optimitzar i depurar els seus programes.

En el següent tema aprofundirem en el tema, veient altres possibilitats amb els programes interpretats i executats en màquines virtuals.

Diferències Clau.

El codi font és llegible i modificable per humans, però necessita ser compilat. El codi objecte és binari i entenedor per la màquina. L’arxiu executable és el resultat final

Codi Font

És el conjunt d'instruccions llegibles per humans. Necessita ser compilat per convertir-se en alguna cosa que la màquina pugui executar. És un arxiu de text.

Codi Objecte

És el resultat intermedi de la compilació del codi font. Conté instruccions binàries que la màquina pot entendre, però no es pot executar directament sense enllaçació addicional.

Arxiu Executable

És el programa final, llest per ser executat per la màquina. Es genera a partir del codi objecte mitjançant el procés d'enllaç.

Exemples de codi

Aquí veurem algunes diferències per a entendre millor els tipus de fitxers.

Exemple de codi font en C

 

    #include <stdio.h>
        void main() {
            printf("Hola, món!\n");
        }

Arxius Codi Font

.c per C
.cpp per C++
.java per Java
.py per Python
.js per JavaScript

Codi Objecte

Quan es compila el codi font d'un programa en C amb el compilador gcc, es genera un fitxer objecte: 

gcc -c programa.c -o programa.o

.o en sistemes Unix/Linux
.obj en sistemes Windows


Fitxers Executables

El fitxer executable és la versió final d'un programa que pot ser executada pel sistema operatiu. Es genera després d'enllaçar el codi objecte amb biblioteques necessàries.

gcc programa.o -o programa.bin

.out o .bin en sistemes Unix/Linux.
.exe en sistemes Windows.
